## 1. 背景与目标

### 1.1 背景

现有系统已具备app_渠系计算前端（明渠/渡槽/隧洞/暗涵/倒虹吸）和水面线推求功能，但缺少**地形建模**和**土石方工程量计算**能力。在渠道工程设计中，需要：

1. 根据地形测量数据（等高线、高程点）建立数字地面模型（TIN）
2. 沿渠道中心线生成纵断面（地面线）和横断面（地形剖面）
3. 将设计断面与地形断面叠加，计算开挖/回填工程量
4. 支持地质分层，分别统计土方和石方

### 1.2 目标

- **Phase 1**（核心算法层）：实现 TIN 建模、断面切割、土石方计算的纯算法库 + 单元测试
- **Phase 2**（UI 集成层）：以新 Tab 模块集成到现有 PySide6 主窗口，提供界面预览和成果输出

### 1.3 用户角色

| 角色 | 描述 |
|------|------|
| 渠道设计工程师 | 主要用户，输入地形和渠道设计参数，获取土石方计算表和图纸 |

## 3. 模块详细设计

### M1：数据导入模块

#### 3.1.1 地形数据导入

| 数据源 | 格式 | 说明 |
|--------|------|------|
| DXF 等高线 | `.dxf` (LWPOLYLINE/POLYLINE/SPLINE) | 读取图层名含"等高线"或用户指定图层，提取多段线及其高程（Z坐标或图层名中的高程值） |
| DXF 高程点 | `.dxf` (POINT/TEXT/INSERT) | 读取离散高程点，高程从Z坐标或关联文字标注获取 |
| 测量坐标文件 | `.csv` / `.txt` | 格式：`点号, X, Y, Z` 或 `X, Y, Z`，支持自定义分隔符 |
| Excel 坐标表 | `.xlsx` | 指定列映射（X列、Y列、Z列、点号列） |

**关键处理逻辑：**
- 等高线离散化：将连续等高线按用户指定间距（默认1m）采样为离散点，保留折点
- 坐标系统一：支持用户指定坐标偏移量（大地坐标→工程坐标）
- 数据清洗：去除重复点、异常高程值过滤
- 多源数据合并：不同来源的点集合并为统一的 `(X, Y, Z)` 点集

#### 3.1.2 渠道中心线导入

| 来源 | 格式 | 说明 |
|------|------|------|
| DXF 多段线 | `.dxf` | 直接导入已绘制的中心线（LWPOLYLINE），提取平面坐标 |
| 桩号+坐标表 | `.xlsx` / `.csv` | 格式：`桩号, X, Y`，按桩号顺序连接 |
| 交点+曲线要素 | 手动输入 / `.xlsx` | 交点坐标 `(X, Y)` + 圆曲线要素（半径R、切线长T），计算生成中心线 |

**桩号系统：**
- 自动沿中心线累计计算桩号里程
- 支持断链处理（桩号跳跃/重叠）
- 中心线上任意桩号 → 平面坐标 `(X, Y)` + 切线方向角的正反查询

#### 3.1.3 地质分层线导入

| 来源 | 格式 | 说明 |
|------|------|------|
| DXF 分层线 | `.dxf` | 按图层区分不同地质层界面线 |
| 横断面上的分层数据 | 手动输入 / `.xlsx` | 在各横断面位置输入各层分界深度 |

#### 3.1.4 设计断面数据来源

- **优先复用**现有「app_渠系计算前端」模块的计算结果（明渠/渡槽/隧洞/暗涵断面参数）
- **允许手动输入/覆盖**：在土石方模块中直接指定断面类型和参数
- 断面参数沿桩号可变（不同桩号段使用不同断面设计）

### M2：TIN 地形建模

#### 3.2.1 三角网构建

**算法选择：约束 Delaunay 三角剖分（CDT）**

- 使用 `triangle` 库实现 CDT
- 等高线作为**约束边**（constraint edges），确保三角网不跨越等高线，地形保真度高
- 高程点作为额外的 Steiner 点加入三角剖分

**为什么必须用 CDT 而非普通 Delaunay？**
> 普通 Delaunay 三角剖分仅考虑点的位置，不保证等高线作为三角形的边。
> CDT 将等高线强制作为三角形边界，确保相邻等高线之间的地形过渡符合实际地貌。
> 这是 AutoCAD Civil 3D 等专业软件采用的标准做法。

**处理流程：**
```
等高线(离散化) + 高程点 → 合并点集 + 约束边集
        ↓
   约束 Delaunay 三角剖分 (triangle 库)
        ↓
   TIN 模型 (三角面片集合，每个顶点带Z值)
        ↓
   建立空间索引 (R-tree / KD-tree，加速查询)
```

#### 3.2.2 高程插值查询

给定平面坐标 `(x, y)`，在 TIN 上查询该点高程：

1. 通过空间索引快速定位所在三角形
2. 在三角形内做重心坐标插值，计算精确高程 Z
3. 查询点在 TIN 范围外时返回 `None`（或最近边界值 + 警告）

**性能目标：** 单点查询 < 0.1ms，批量查询（10万点）< 1s

#### 3.2.3 地质分层 TIN

- 为每个地质分层界面单独建立 TIN（或使用分层线插值）
- 在横断面切割时，同时切出各层界面线
- 支持简化模式：用户仅输入各桩号处的分层深度，线性插值

#### 3.2.4 性能优化策略（大规模数据）

| 数据规模 | 策略 |
|----------|------|
| < 10万点 | 直接全量构建，无需特殊处理 |
| 10万~100万点 | 分块构建 + 边界缝合，使用空间索引加速 |
| > 100万点 | 渠道缓冲区裁剪（仅处理中心线两侧一定范围内的数据） |

### M3：断面切割

#### 3.3.1 纵断面生成

沿渠道中心线生成地面纵断面线：

1. 沿中心线按指定步长（默认 1m）等距采样点
2. 每个采样点查询 TIN 高程
3. 得到 `(桩号, 地面高程)` 序列，即纵断面地面线
4. 同时获取设计渠底高程线（来自纵坡设计参数或用户输入）

**输出：**
- 纵断面数据表：`桩号 | 地面高程 | 设计底高程 | 挖深/填高`
- 纵断面预览图（matplotlib）

#### 3.3.2 横断面切割

在指定桩号位置，垂直于中心线方向切割地形：

**断面生成规则：**
- 按固定间距（用户可配置，默认 20m）沿中心线自动生成
- 允许在关键位置（弯道起终点、建筑物衔接处、地形突变处）手动添加额外断面
- 断面线方向：垂直于该点的中心线切线方向

**断面宽度确定：**
- 自动模式：根据设计断面的开口宽度 × 系数（默认 2.0）+ 边坡延伸 + 余量
- 手动覆盖：用户可单独指定每个断面的左右宽度（支持不对称）
- 采样间距：横断面上按 0.5m（默认）等距采样 TIN 高程

**每个横断面包含：**
```
        ← 左侧                    右侧 →
  ┌──────────────────────┬──────────────────────┐
  │    地面线 (TIN切割)    │    地面线 (TIN切割)    │
  │    设计断面轮廓线      │    设计断面轮廓线      │
  │    地质分层界面线      │    地质分层界面线      │
  │    开挖区域 (填充)     │    开挖区域 (填充)     │
  │    回填区域 (填充)     │    回填区域 (填充)     │
  └──────────────────────┴──────────────────────┘
         距中心线距离 →       ← 距中心线距离
```

#### 3.3.3 断面面积计算

每个横断面自动计算：
- **开挖面积** A_挖 = 地面线在设计线以上围成的面积
- **回填面积** A_填 = 设计线在地面线以上围成的面积
- **分层面积**（有地质分层时）：各层的开挖面积分别统计
- 面积计算方法：多边形面积公式（Shoelace formula）

### M4：工程量计算与输出

#### 3.4.1 体积计算方法

支持三种方法，用户可选择并对比：

**方法一：平均断面法**
```
V = Σ [(A_i + A_{i+1}) / 2 × L_i]
```
- `A_i`, `A_{i+1}`：相邻两个断面的面积
- `L_i`：两断面间的中心线距离
- 水利行业最常用，计算简单

**方法二：棱台法**
```
V = Σ [L_i / 6 × (A_i + A_{i+1} + 4×A_m)]
```
- `A_m`：中间断面面积（桩号中点处插值计算）
- 精度优于平均断面法，适用于面积变化大的渠段

**方法三：TIN 体积法**
```
V = Σ |V_prism|    (三角棱柱体积累加)
```
- 地面 TIN 与设计面 TIN 之间直接做三维体积差
- 精度最高，不依赖断面间距
- 计算量较大，适合精确校核

#### 3.4.2 土石方分类统计

结合地质分层数据：
- 在每个横断面上，计算各地质层的开挖面积
- 按方法一/方法二分别计算各层体积
- 汇总表中分列：**普通土方** / **硬土方** / **软石方** / **硬石方**（或用户自定义类别）

#### 3.4.3 输出成果

**Excel 表格输出：**

| 表格名称 | 内容 |
|----------|------|
| 纵断面数据表 | 桩号、地面高程、设计底高程、挖深/填高 |
| 横断面面积汇总表 | 桩号、开挖面积(分层)、回填面积、断面宽度 |
| 土石方工程量计算表 | 桩号段、开挖方量(分层)、回填方量、累计方量 |
| 工程量汇总表 | 按渠段/全线汇总的土方、石方、回填方总量 |
| 方法对比表 | 三种计算方法的结果对比及差异百分比 |

**DXF 图纸输出：**

| 图纸名称 | 内容 |
|----------|------|
| 纵断面图 | 地面线 + 设计底线 + 桩号标注 + 挖深标注 |
| 横断面图（批量） | 每个桩号的横断面图：地面线 + 设计断面 + 面积标注 |

**软件界面预览：**
- 纵断面交互式预览（缩放、平移、鼠标悬停显示数据）
- 横断面逐个浏览/翻页，实时显示面积和分层
- TIN 三维预览（可选，使用 matplotlib 3D 或 PyVista widget）

### 4.5 第三方库调研结论

> 经过系统搜索（2026-02-22），Python 生态中**不存在**集成了「TIN建模 + 中心线 + 断面切割 + 土石方」的一站式开源库。
> 最接近的两个库：
> - **`startinpy`**：Rust 实现的 2.5D 地形 Delaunay + 多种插值（自然邻域/Laplace/IDW），性能极好，但**不支持 CDT**（约束边）
> - **`volpy`**：TIN 体积计算，但功能单一且不活跃（2019年停更）
>
> **技术策略**：用 `triangle` 做 CDT 构建（等高线约束边），用 `startinpy` 做高精度插值查询，其余模块（中心线/断面切割/土石方/DXF输出）自研。

## 8. 风险与约束

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| DXF 等高线高程提取不一致 | 数据无法正确导入 | 支持多种高程来源（Z坐标/图层名/文字标注），提供预览确认 |
| 大规模数据性能瓶颈 | 用户体验差 | 渠道缓冲区裁剪 + 分块构建 + 异步处理 |
| CDT 约束边自交叉 | 三角剖分失败 | 数据预处理：检测并修复自交叉等高线 |
| 地质分层数据不完整 | 土石方分类不准 | 支持退化为"按比例分配"模式 |
| 曲线段中心线精度 | 桩号/方向角误差 | 曲线段加密采样，圆曲线精确公式计算 |

### 分级放坡数据结构（已确认）

```python
@dataclass
class SlopeGrade:
    """单级边坡参数
    
    示意图（从渠底向上看）：
    
          马道(berm_width)
         ___________
        |           |
        |           | ← 本级高度(height)
        |  坡比     |
        | (ratio)   |
        |___________|
    """
    ratio: float          # 边坡比（如 1.0 表示 1:1，0.75 表示 1:0.75）
    height: float         # 本级高度（m），最后一级可设为 inf 表示延伸到地面
    berm_width: float     # 本级顶部马道宽度（m），0 表示无马道

@dataclass
class ExcavationSlope:
    """某桩号段的开挖边坡配置"""
    start_station: float  # 起始桩号（m）
    end_station: float    # 终止桩号（m）
    left_grades: list[SlopeGrade]   # 左侧分级边坡（从渠底向上）
    right_grades: list[SlopeGrade]  # 右侧分级边坡（从渠底向上）
    platform_enabled: bool = False  # 是否在坡顶预留施工便道
    platform_width: float = 2.0    # 施工便道宽度（m）
```

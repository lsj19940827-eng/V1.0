# 面板合并 v3.1 验收清单（可直接执行）

> 适用分支：`codex/merged-panel-v31`  
> 验收目标：确认“断面批量计算 + 推求水面线”合并链路可用、可保存、可回滚

---

## 一、阶段一验收（可算可用）

### 场景1：全成功链路
- [ ] 进入“推求水面线”面板，表1填写至少 2 行有效断面参数。
- [ ] 点击“断面批量计算”。
- [ ] 预期：表2生成结果且无失败行；表3自动同步；“插入渐变段/倒虹吸/有压管道/执行计算”按钮解锁。
- [ ] 预期状态文字：`状态：断面全成功，表1+表2已同步到表3`。

### 场景2：失败锁定链路
- [ ] 在表1制造至少 1 行非法参数（如必填项留空或数值越界）。
- [ ] 点击“断面批量计算”。
- [ ] 预期：表2出现失败提示；下游按钮保持锁定；出现“存在失败行，已锁定...”提示。

### 场景3：编辑失效机制
- [ ] 先完成“场景1”得到已解锁状态。
- [ ] 编辑表1任意单元格（任意行任意列）。
- [ ] 预期：立即回到锁定状态；状态文字变为“表1已变更，结果已失效，请重新执行断面批量计算”。

### 场景4：入口收敛
- [ ] 检查左侧导航。
- [ ] 预期：不再有“批量计算”独立入口，仅保留“推求水面线”。

### 场景5：下游门禁
- [ ] 在“未同步/失效”状态点击“插入渐变段/倒虹吸/有压管道/执行计算”。
- [ ] 预期：弹出“操作已锁定”提示，不执行下游动作。
- [ ] 在“已同步”状态重复点击上述按钮。
- [ ] 预期：可正常进入各子流程。

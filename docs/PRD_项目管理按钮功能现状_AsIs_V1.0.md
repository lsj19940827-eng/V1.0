1. `项目管理`按钮可见功能（菜单项、快捷键、最近项目）

## 5. 功能需求明细（As-Is）

## 5.1 PM-01 新建项目

### 触发方式

1. 菜单点击 `📄 新建项目`
2. 快捷键 `Ctrl+N`

### 前置规则

1. 若当前存在未保存修改（`is_dirty=True`），先弹出三按钮对话框：`保存/放弃/取消`
2. 用户选择：
   - `保存`：先执行保存流程，保存成功后继续新建
   - `放弃`：直接继续新建
   - `取消`：终止新建

### 执行结果

1. 当前项目路径置空（表示“新项目未保存”）
2. 脏状态清除
3. 发射 `project_changed("")`
4. 状态栏消息：`已新建项目`

### 数据清空范围（重要）

当前代码仅明确重置以下面板：

1. 批量计算面板（输入表、渠道名、起始水位、起始桩号、级别、批量结果）
2. 推求水面线面板（节点、计算节点、表格、渠道名、水位、流量、桩号、糙率、级别）

其余面板（明渠/渡槽/隧洞/暗涵/倒虹吸/有压管道）未在 `new_project()` 中显式清空。

---

## 5.2 PM-02 打开项目

### 触发方式

1. 菜单点击 `📂 打开项目...`
2. 快捷键 `Ctrl+O`
3. 最近项目子菜单点击某一项（带路径参数）

### 前置规则

与新建一致：若脏状态为真，先执行“保存/放弃/取消”确认。

### 文件选择规则

1. 若未传入路径，弹出文件选择框
2. 过滤器：`渠系项目文件 (*.qxproj)`

### 打开流程

1. 校验路径存在，不存在则提示 `文件不存在` 并中止
2. 读取 JSON，调用各面板 `from_project_dict`
3. 更新当前路径 `_current_path`
4. 清除脏状态
5. 写入最近项目列表
6. 发射 `project_changed(path)`
7. 状态栏消息：`已打开项目: {文件名}`

### 失败反馈

1. JSON读取或恢复失败：弹出 `加载失败` 错误框

---

## 5.3 PM-03 保存项目

### 触发方式

1. 菜单点击 `💾 保存项目`
2. 快捷键 `Ctrl+S`

### 规则

1. 若当前项目尚无路径（新项目），自动转入“另存为”流程
2. 有路径时直接覆盖保存

### 执行结果

1. 收集所有面板项目数据并写入 `.qxproj`
2. 清除脏状态
3. 状态栏消息：`已保存: {文件名}`

### 失败反馈

1. 弹出 `保存失败` 错误框

---

## 5.4 PM-04 另存为

### 触发方式

1. 菜单点击 `📥 另存为...`
2. 快捷键 `Ctrl+Shift+S`
3. 保存时无当前路径自动进入

### 规则

1. 默认文件名自动生成：`渠道名_级别_时间戳.qxproj` 或 `渠道名_时间戳.qxproj`
2. 若用户未输入 `.qxproj` 后缀，系统自动补全
3. 保存成功后将该路径设为当前项目路径
4. 写入最近项目

### 默认文件名来源

1. 优先取“批量计算面板”的渠道名与级别
2. 批量面板不可用时才回退到水面线面板

---

## 5.5 PM-05 最近项目

### 数据规则

1. 最大数量：5
2. 去重策略：同路径先移除再插入队首
3. 持久化：`QSettings("ShuiFa", "QuxiDesign")` 下 `recent_projects`
4. 启动加载时过滤不存在路径

### UI规则

1. 无数据：显示 `(无最近项目)` 且禁用
2. 有数据：显示文件名（`basename`），完整路径在 tooltip
3. 菜单底部固定 `清空最近项目`

### 清空动作

1. 点击 `清空最近项目`
2. 内存列表清空并写回 `QSettings`
3. 立即刷新子菜单

---

## 5.6 PM-06 脏状态管理

### 状态定义

- `is_dirty=False`：无未保存改动
- `is_dirty=True`：有未保存改动

### 置脏来源

项目管理器连接了8个面板的 `data_changed` 信号：

1. 批量计算
2. 推求水面线
3. 明渠
4. 渡槽
5. 隧洞
6. 暗涵
7. 倒虹吸
8. 有压管道

### 清脏时机

1. 保存成功
2. 另存为成功
3. 打开项目成功
4. 新建项目成功

---

## 5.7 PM-07 自动保存

### 触发规则

1. 主窗口初始化项目管理器后启动定时器
2. 间隔：`60,000 ms`（1分钟）
3. 仅当 `is_dirty=True` 时执行

### 自动保存位置

- `{程序根目录}/data/`

### 自动保存文件名

- `{safe_name}_{safe_level}_{YYYYMMDD_HHMMSS}_autosave.qxproj`
- `safe_name` 最长20字符，`safe_level` 最长10字符
- 仅保留字母数字、空格、下划线、连字符（中文在 Python `isalnum()` 下属于可保留字符）

### 执行结果

1. 成功：状态栏提示 `自动保存完成: {文件名}`
2. 失败：状态栏提示 `自动保存失败: {异常}`
3. 自动保存不清除脏状态

---

## 5.8 PM-08 关闭窗口前保存确认

### 触发点

- 主窗口 `closeEvent`

### 规则

1. 先调用 `check_save_on_close()`
2. 若脏状态为真，弹出“保存/放弃/取消”
3. `取消`：阻止关闭
4. `保存`：执行保存成功后继续关闭，失败则中止关闭
5. `放弃`：直接关闭

---

## 5.9 PM-09 窗口标题联动

### 标题格式

- `{项目文件名或“新项目”}{可选星号} - {系统基础标题}`
- 有未保存变更时显示 `*`

### 联动时机

1. 项目路径变化（新建/打开/另存为）
2. 脏状态变化

## 6. 项目文件格式规范（As-Is）

## 6.1 文件基础

1. 扩展名：`.qxproj`
2. 编码：UTF-8
3. 存储格式：JSON（`ensure_ascii=False`、`indent=2`）

## 6.2 顶层结构

```json
{
  "version": "1.1",
  "created_at": "2026-03-04T10:11:12.123456",
  "batch_panel": { ... } | null,
  "water_profile_panel": { ... } | null,
  "open_channel_panel": { ... } | null,
  "aqueduct_panel": { ... } | null,
  "tunnel_panel": { ... } | null,
  "culvert_panel": { ... } | null,
  "siphon_panel": { ... } | null,
  "pressure_pipe_panel": { ... } | null
}
```

说明：`version`/`created_at` 当前仅写入，不参与版本迁移判断。

## 6.3 各面板数据覆盖（现状）

### A. 批量计算 `batch_panel`

1. 基础信息：`channel_name`、`channel_level`、`start_water_level`、`start_station`
2. 流量段：`flow_segments`
3. 选项：`inc_checked`、`detail_checked`
4. 输入表：`input_rows`（21列*N行）
5. 加载后会清空历史计算结果，需重新计算

### B. 推求水面线 `water_profile_panel`

1. `ui_settings`：渠道、流量、桩号、糙率、转弯半径、渐变段参数等
2. `project_settings`：`ProjectSettings`序列化结果
3. `nodes`：原始节点
4. `calculated_nodes`：计算后节点
5. `extra_caches`：结构高程、倒角、U形参数、文字导出设置、平面图文字设置、自定义有压管参数等
6. `siphon_roughness_data`、`pressure_pipe_roughness_data`

### C. 明渠/渡槽/隧洞/暗涵

四者结构基本一致：

1. `cases`：工况集合
2. `current_case_idx`：当前工况索引
3. `all_results`：全部工况结果
4. `current_result`：当前结果
5. `input_params`：输入参数缓存
6. `notebook_idx`：当前页签

### D. 倒虹吸 `siphon_panel`

通过 `to_dict()/from_dict()` 复用项目序列化，主要包含：

1. 水力参数（流量、糙率、流速策略、局损系数等）
2. 管段与平面段数据（`segments/plan_segments`）
3. 平面特征点、纵断面节点
4. 计算结果关键值（如总水头损失、管径、流速）
5. 多项兼容旧键名的恢复逻辑

### E. 有压管道 `pressure_pipe_panel`

1. `cases`
2. `current_case_idx`
3. `last_errors`
4. `notebook_idx`

说明：加载后结果集合会重置，错误日志可恢复显示。

## 7. 交互反馈与异常处理

## 7.1 对话框

1. 保存确认：自定义 Fluent 风格三按钮（保存/放弃/取消）
2. 打开/另存：系统文件对话框
3. 严重错误：`QMessageBox.critical`
4. 一般错误（文件不存在）：`QMessageBox.warning`

## 7.2 状态栏反馈

项目管理器通过 `status_message` 信号发送文本，主窗口统一显示5秒：

1. `已新建项目`
2. `已打开项目: ...`
3. `已保存: ...`
4. `自动保存完成: ...`
5. `自动保存失败: ...`

## 8. 非功能与实现约束

1. 不依赖网络，仅本地文件读写
2. 最近项目与项目设置分属不同 `QSettings` 命名空间
3. 自动保存文件默认不清理，目录可能随使用增长
4. 保存采用整文件重写，不是增量写入
5. 读写异常通过UI提示，面板序列化异常多采用控制台打印后继续

---

## 9. 当前已知行为与边界（需认知，不是缺陷判定）

1. `新建项目`只显式清空“批量计算/推求水面线”两块，其余模块未统一重置。
2. 自动保存与另存为默认命名优先读取“批量计算面板”渠道信息，不随当前活动模块变化。
3. 顶层字段名 `created_at` 在每次保存都会更新，语义更接近“本次写盘时间”。
4. 加载项目时仅恢复存在且为真值的模块字段；若某模块字段缺失，可能保留当前内存中的旧状态。
5. 最近项目“路径存在性过滤”发生在启动加载时；程序运行中若文件被外部删除，列表项仍可能暂时存在，点击时才报不存在。
6. `⚙ 项目设置`数据保存在独立 `QSettings("SichuanShuifa", "HydroCalc")`，不等同于 `.qxproj` 的统一项目载入行为。

---

## 10. 现状验收清单（可直接测试）

1. 修改任一已接入面板数据后，窗口标题出现 `*`。
2. 点击 `保存项目` 后 `*` 消失。
3. 新建/打开/关闭窗口时，若有未保存修改，均出现保存确认三按钮。
4. `Ctrl+N/O/S`、`Ctrl+Shift+S` 与菜单行为一致。
5. `另存为` 未写后缀时自动补 `.qxproj`。
6. 打开成功后最近项目列表更新，最多保留5项。
7. 1分钟内有脏数据时会在 `data/` 目录生成 `_autosave.qxproj` 文件。
8. 打开不存在路径时弹出文件不存在提示，不崩溃。

## 一、模块概述

矩形暗渠（矩形暗涵）水力计算模块，为"渠系建筑物水力计算系统"的子模块之一，提供矩形封闭断面暗涵的水力设计计算功能。

### 1.1 定位

- 在主窗口侧边导航栏中以 **"矩形暗涵设计"** 项出现，索引位置为第 4 个模块（0-indexed: 3）
- 导航提示文字：`经济最优断面/指定参数`

### 1.2 核心能力

| 能力 | 说明 |
|------|------|
| 全局经济最优断面自动搜索 | 当底宽、宽深比和高宽比均留空时，以 β 为搜索变量，求解 B×H 最小的断面（β 无硬约束） |
| 指定底宽计算 | 用户指定底宽 B，系统搜索满足约束的最优断面高度 H |
| 指定宽深比计算 | 用户指定宽深比 β=B/h，系统搜索最优 B 和 H |
| **指定高宽比计算** | 用户指定高宽比 H/B，H=(H/B)×B，系统搜索满足约束的最小 B |
| 加大流量工况校核 | 自动或指定加大流量百分比，校核净空和流速 |
| 净空约束验证 | 按规范要求验证净空高度和净空面积百分比 |
| 断面图预览 | matplotlib 绘制设计/加大两种工况的断面对比图 |
| 导出 DXF | 带比例尺、图层、标注的矩形暗涵断面工程图 |
| 导出 Word | 高端咨询报告风格的计算书（含可编辑 OMML 公式） |

## 三、计算内核（矩形暗涵设计.py）

### 3.1 常量定义

| 常量 | 值 | 说明 |
|------|----|------|
| `MIN_WIDTH_RECT` | 0.2 m | 最小宽度 |
| `MIN_HEIGHT_RECT` | 0.5 m | 最小高度 |
| `BH_RATIO_MIN_RECT` | 0.5 | 推荐宽深比下限 (B/h) |
| `BH_RATIO_MAX_RECT` | 2.5 | 推荐宽深比上限 (B/h) |
| `OPTIMAL_BH_RATIO` | 2.0 | 保留用于向后兼容（不再为默认搜索目标） |
| `HB_RATIO_LIMIT` | 1.2 | 高宽比/宽高比双向建议阈值（H/B>1.2 或 B/H>1.2 时输出 ⚠ 提醒，**不强制拒绝**） |
| `MIN_FREEBOARD_PCT_RECT` | 0.10 (10%) | 最小净空面积百分比 |
| `MAX_FREEBOARD_PCT_RECT` | 0.30 (30%) | 最大净空面积百分比 |
| `MIN_FREEBOARD_HGT_RECT` | 0.4 m | 最小净空高度 |
| `SOLVER_TOLERANCE` | 0.0001 | 求解器收敛容差 |
| `MAX_ITERATIONS` | 100 | 最大迭代次数 |
| `DIM_INCREMENT` | 0.01 m | 尺寸搜索步长 |

### 3.2 加大流量百分比查表

函数: `get_flow_increase_percent(design_Q)`

| 设计流量 Q (m³/s) | 加大比例 |
|--------------------|---------|
| Q < 1 | 30% |
| 1 ≤ Q < 5 | 25% |
| 5 ≤ Q < 20 | 20% |
| 20 ≤ Q < 50 | 15% |
| 50 ≤ Q < 100 | 10% |
| Q ≥ 100 | 5% |

用户可指定加大流量百分比覆盖自动计算值。

### 3.3 净空高度要求

函数: `get_required_freeboard_height_rect(H_total)`

根据 GB 50288-2018 第11.2.5条 表11.2.5（无压涵洞的净空高度）：

- 在任何情况下，净空高度均不得小于 **0.4m**
- 矩形涵洞进口净高 D ≤ 3m 时，净空高度应不小于 **D/6**
- 矩形涵洞进口净高 D > 3m 时，净空高度应不小于 **0.5m**

实现逻辑：
```
if H ≤ 3.0:  return max(0.4, H/6)
else:         return 0.5
```

### 3.4 水力要素计算

函数: `calculate_rectangular_outputs(B, H, h, n, slope)`

**输入**:
- `B`: 底宽 (m)
- `H`: 涵洞内侧总高度 (m)
- `h`: 水深 (m)，自动截断为 min(h, H)
- `n`: 糙率
- `slope`: 水力坡降 i

**公式**:
- 过水面积: `A = B × h`
- 湿周: `P = B + 2h`（矩形封闭断面，水面不计湿周）
- 水力半径: `R = A / P`
- 流量 (曼宁公式): `Q = (1/n) × A × R^(2/3) × i^(1/2)`
- 流速: `V = Q / A`
- 总断面面积: `A_total = B × H`
- 净空面积百分比: `freeboard_pct = (A_total - A) / A_total × 100%`
- 净空高度: `freeboard_hgt = H - h`

**输出**: 字典，键为 `A`, `P`, `R_hyd`, `V`, `Q`, `A_total`, `freeboard_pct`, `freeboard_hgt`

### 3.5 水深反算（二分法）

函数: `solve_water_depth_rectangular(B, H, n, slope, Q_target)`

- 搜索范围: `[0.00001, H]`
- 方法: 二分法，最大 100 次迭代
- 收敛判据: `|Q_mid - Q_target| / Q_target < 0.0001`
- 返回: `(水深, 是否成功)` 元组

### 3.6 主计算函数

函数: `quick_calculate_rectangular_culvert(...)`

**输入参数**:

| 参数 | 类型 | 说明 |
|------|------|------|
| `Q` | float | 设计流量 (m³/s) |
| `n` | float | 糙率 |
| `slope_inv` | float | 水力坡降倒数 (1/i) |
| `v_min` | float | 最小流速 (m/s) |
| `v_max` | float | 最大流速 (m/s) |
| `target_HB_ratio` | float, 可选 | 目标高宽比 H/B（指定后 H=(H/B)×B，搜索满足约束的最小B） |
| `target_BH_ratio` | float, 可选 | 目标宽深比 B/h |
| `manual_B` | float, 可选 | 指定底宽 (m) |
| `manual_increase_percent` | float, 可选 | 指定加大流量百分比 |

**设计模式判定逻辑**:

```
if manual_B 未指定 AND target_BH_ratio 未指定 AND target_HB_ratio 未指定:
    → 经济最优断面模式（β 参数化，B×H 最小）
elif target_HB_ratio 已指定:
    → 指定高宽比模式：对每个B，H=target_HB_ratio×B，搜索满足约束的最小B
elif target_BH_ratio 已指定:
    → 目标宽深比模式
elif manual_B 已指定:
    → 指定底宽模式
注：target_BH_ratio 与 target_HB_ratio 不可同时指定（过约束）
```

**搜索算法**:

#### 模式一：全局经济最优断面（β 参数化，v1.2 重写）

**原理**：明渠 β=2 对暗涵不是最优——H/B 建议值曾在 β 偏大时把 H 强行拉高，导致总面积 B×H 增加（v1.7已移除该硬约束）。新算法以 β 为搜索变量，解析计算 h_design 和 B，全局取面积最小。

**解析公式（设计工况，无需迭代）**：
```
h_design(β) = (Q·n/√i · (β+2)^(2/3) / β^(5/3))^(3/8)
B(β) = β · h_design(β)
```

**搜索流程**：
1. **阶段1**：粗扫 β ∈ [`BH_RATIO_MIN_RECT`, `BH_RATIO_MAX_RECT`]=[0.5, 2.5]，步长 0.02
2. 对每个 β：
   - 解析计算 h_design 和 B（无迭代）
   - 流速约束：`v_min ≤ V ≤ v_max`，`B ≥ 0.2m`
   - 热启动割线法求 h_inc（利用相邻 β 的 h_inc 作初值；失败回退带括号二分）
   - 解析求 H_min（净空面积≥10%、净空高度≥0.4m/H/6 取 max）；H_max（净空面积≤30%）
   - 约束验证：加大工况净空严格 [10%, 30%]；设计工况只检下限≥10%（无上限）
   - H/B、B/H 建议值（≤1.2）不参与 H_min/H_max 计算，仅在结果中标注 ⚠
   - 全量记录最小 B×H（不提前 break）
3. **阶段2**：在粗扫最优 β 附近 ±0.06，步长 0.002 细化
4. 若粗扫无解，扩展至 β ∈ [0.3, 3.5] 重试
5. 仍无解则退回普通搜索模式

#### 模式二：普通搜索（指定底宽/指定宽深比/全范围搜索）

1. B 范围：指定底宽时固定，否则 0.2~20.0
2. 宽深比范围：指定时 `[target±0.2]`，否则 `[0.5, 2.5]`
3. 双重循环遍历 B × β 组合
4. 同样的约束检查
5. 优选标准：
   - 有目标 β 时：β 偏差最小优先
   - 无目标 β 时：总面积最小优先

### 4.5 结果显示

#### 简要模式（detail_cb 未选中）

分为4个区块：
1. **输入参数** — Q, n, i, v_min, v_max
2. **断面尺寸** — B, H, β, H/B（经济最优断面标注★）
3. **设计流量工况** — h, V, Fb, 净空%
4. **加大流量工况**（仅勾选时显示） — 加大比例, Q加大, h加大, V加大, Fb加大, 净空%
5. **验证结果** — 流速✓/✗, 净空✓/✗

**加大流量条件化显示**：当 `use_increase=False`（不勾选“考虑加大流量比例系数”）时，简要/详细模式均跳过加大流量工况段落，净空验证和综合验证使用设计流量工况数据，章节编号动态调整（四→净空验证，五→综合验证）。

#### 详细模式（detail_cb 选中）

分为6个章节（勾选加大流量时），共18步计算过程：
1. **一、输入参数**
2. **二、断面尺寸** — 宽深比计算、高宽比计算、总断面积计算
3. **三、设计流量工况** — 步骤4~10：过水面积、湿周、水力半径、流速(曼宁)、流量验证、净空高度、净空面积比
4. **四、加大流量工况**（仅勾选时） — 步骤11~18：加大流量、加大水深、加大面积/湿周/半径/流速/流量校核/净空
5. **五、净空验证** — 面积验证 + 高度验证（引用规范条文）
6. **六、综合验证** — 流速/净空面积/净空高度三项汇总

不勾选时章节编号动态调整为：一→二→三→四（净空验证）→五（综合验证）。

渲染方式: `plain_text_to_formula_html()` → SVG公式 + `QWebEngineView`

## 五、DXF 导出（dxf_export.py）

### 5.1 导出入口

## 九、设计决策记录

### D1: 经济最优断面为默认模式
当底宽和宽深比均留空时，系统默认搜索总面积 B×H 最小的断面（β 无硬约束）。明渠 β=2 对带顶板的暗涵不是最优：H/B≤1.2 的结构约束会在 β 过大时将 H 强行拉高，浪费截面，实测可节省约 10~15% 材料。

### D2: 高宽比单向限制（仅限 H/B）
`H/B ≤ 1.2`，限制过窄断面（H过大），参考 GB 50288-2018 第11.2.5条表11.2.5。**不限制 B/H**（宽浅断面在工程中合理存在，如 B=2.4m、Q=3m³/s 的场景）。v1.6修复：移除了原代码中`B/H > 1.2`的错误双向拦截，该约束不来自规范，导致宽浅断面合法方案被误拒。

### D3: 搜索失败退化策略
经济最优断面搜索失败时，自动退回全范围普通搜索模式，而非直接报错。

### D4: 净空验证关键约束在加大流量工况
勾选加大流量时，关键约束在加大流量工况（10%~30% 严格）；设计流量工况只检下限≥10%，不检上限（H 由加大水深决定，设计净空自然偏大）。不勾选时，设计流量工况严格 10%~30%。v1.6修复：移除原代码中设计流量上限`+20%`的错误放宽。

### D5: 暗涵为封闭断面
与明渠的开放断面不同，暗涵绘图时顶部为实线+填充标记，表示封闭结构。

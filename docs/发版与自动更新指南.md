# 发版与自动更新指南

## 一、整体架构

```
用户端（应用内）                       更新源
┌──────────────┐  ① 优先局域网     ┌──────────────────┐
│  启动时静默检查  │ ──────────▶ │  \\IP\CanalUpdate  │
│  或手动点击按钮  │                │  version.json      │
│                │  ② 回退GitHub     │  + zip 文件        │
│                │ ──────────▶ └──────────────────┘
└──────┬───────┘        │       ┌──────────────────┐
       │ 发现新版本        └────▶ │  Gist/version.json │
       ▼                         │  GitHub Releases   │
┌──────────────┐                 └──────────────────┘
│  下载更新包     │
│  局域网: 文件复制 │  ← 极快（内网速度）
│  外网: HTTP下载 │  ← 回退方案
└──────┬───────┘
       ▼
┌──────────────┐
│  .bat 脚本执行  │
│  补丁: 覆盖变化  │
│  全量: 备份+解压 │
│  启动新程序     │
└──────────────┘
```

## 二、发版流程（一键自动）

### 方式 A：一键发版（推荐）

```bash
# 补丁版本 1.0.2 → 1.0.3
python tools/release.py patch

# 次版本 1.0.2 → 1.1.0
python tools/release.py minor

# 主版本 1.0.2 → 2.0.0
python tools/release.py major

# 带更新日志
python tools/release.py patch -m "- 修复水面线计算bug\n- 新增批量导出功能"
```

**一条命令自动完成全部 7 步**：
1. 递增 `version.py` 版本号
2. PyInstaller 打包（全量包 + 增量补丁包）
3. Git commit + tag + push
4. 创建 GitHub Release
5. 上传 zip 到 Release Assets
6. 更新 Gist `version.json`
7. **同步到局域网共享文件夹**（同事无需外网即可更新）

另有 GUI 版本：`python tools/release_gui.py`

### 方式 B：手动分步

如果网络有问题或需要更精细控制，可以分步执行：

```bash
# 1. 递增版本号 + 打包
python tools/build.py --bump patch

# 2. 手动 git commit + tag + push
git add version.py
git commit -m "release: v1.0.3"
git tag v1.0.3
git push origin main && git push origin v1.0.3

# 3. 手动到 GitHub 创建 Release，上传 dist/ 下的 zip
# 4. 手动编辑 Gist version.json
```

### version.json 字段说明

| 字段 | 说明 |
|------|------|
| `latest_version` | 最新版本号（与 `version.py` 一致） |
| `download_url` | 全量包直接下载地址 |
| `patch_url` | 增量补丁包下载地址（可选） |
| `changelog` | 更新日志（`\n` 分隔，`- ` 开头会渲染为列表） |
| `release_date` | 发布日期 |
| `min_version` | 最低兼容版本（低于此版本会提示"强制更新"） |
| `file_size_mb` | 全量包大小（用于UI展示） |
| `patch_size_mb` | 补丁包大小（可选） |

## 四、用户端体验

### 自动检查
- 程序启动后 **后台静默检查**（不阻塞界面）
- 有新版本时，右上角弹出 InfoBar 提示
- 状态栏显示 "✨ 新版本 Vx.x.x 可用！"

### 手动检查
- 点击侧边栏 **「🔄 检查更新」** 按钮
- 弹出对话框，显示更新日志（显示补丁包/全量包大小）
- 点击 **「下载并更新」** → **优先下载补丁包**，失败时自动回退全量包
- 进度条 → **「立即安装更新」**
- 确认后：程序关闭 → bat 脚本替换文件 → 自动重启

### 更新安全
- **补丁模式**：只覆盖变化文件，不删除旧文件（除非 patch_manifest.json 指定了删除列表）
- **全量模式**：旧版本自动备份到 `_backup/`，失败可手动恢复
- 授权文件 `.lic` 永远不会被覆盖

## 五、增量补丁原理

### 打包时
1. PyInstaller 打包完成后，扫描所有文件生成 `manifest-Vx.x.x.json`（每个文件 → SHA256）
2. 如果 `dist/` 下存在旧版 manifest，自动对比新旧两个 manifest
3. 将新增/修改的文件打成 `-patch.zip`，内含 `patch_manifest.json` 记录变化详情

### 客户端
1. 优先下载 `patch_url`（通常 3~10MB）
2. 补丁包下载失败 → 自动回退下载 `download_url`（全量包 ~300MB）
3. 补丁模式：直接 xcopy 覆盖变化文件 + 根据删除列表清理
4. 全量模式：备份全部旧文件 → 解压覆盖 → 恢复 .lic

### 重要：保留 manifest 文件
每次打包后，`dist/manifest-Vx.x.x.json` **必须保留**。下次打包时会自动与它对比生成补丁。如果删除了，则下次只能生成全量包。

## 七、更新源优先级

客户端检查更新时：

| 优先级 | 源 | 适用场景 | 速度 |
|--------|------|----------|------|
| ① | 局域网共享 `\\IP\CanalUpdate` | 同事无外网 | 极快（内网复制） |
| ② | GitHub Gist + Releases | 有外网 | 取决于网速 |

- 局域网源不可用时自动回退到 GitHub
- 两个源都不可用时静默跳过（不影响正常使用）
- 如果 GitHub 也不需要，可以只用局域网共享文件夹做唯一更新源
